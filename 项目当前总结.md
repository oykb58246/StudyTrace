## 一、项目定位

StudyTrace（学迹）是面向大学生学习场景的 Flutter 移动端应用。项目以课程任务、学习日志、日历、番茄钟、学习笔记、知识闪卡、学习数据看板、云端同步、学习小组和排行榜 UI 为基础，结合蓝心大模型与可选 DeepSeek 接入，形成「记录—执行—分析—复盘」的学习闭环。

项目参加 **第三届（2026）中国高校计算机大赛 - AIGC 创新赛应用赛道**。

当前版本已加入自建后端骨架：Node/NestJS + PostgreSQL + Prisma + JWT，用于上线后的账号、云同步、AI 代理、学习小组和排行榜。

## 四、核心功能状态

### 已完成的 App 功能

- 任务管理：新增、编辑、删除、状态切换、搜索、筛选、子任务、截止时间、提醒时间。
- 学习日志：新增、删除、搜索、课程筛选。
- 学习日历：日期 marker 同时展示日志和截止任务，点击日期展示当天任务与学习记录，默认展示当天信息。
- 数据看板：任务/日志/周报/笔记统计，子任务进度，近 7 天柱状图，近 4 周趋势，课程分布饼图和课程明细。
- 学习笔记：Notion 风格块编辑。
- 专注计时：番茄钟预设时长，完成后可生成 AI 学习记录。
- 知识闪卡：横向滑动 shelf、小卡层叠、放大浏览、翻转、收藏、标签。
- 云端同步 UI：提供同步入口、同步状态和多端同步相关界面，等待后端接口联调。
- 学习小组 UI：提供邀请制小组、成员列表、组内动态等多人学习场景界面。
- 排行榜 UI：提供个人榜、小组周榜/月榜等学习积分排行展示界面。
- AI 学习助手：生成日志、任务拆解、周报分析、风险提醒。
- AI 对话：流式响应、Markdown 渲染、会话历史、多会话切换。
- AI 设置：蓝心模型配置、DeepSeek 可选配置、高级参数、测试连接。
- 系统设置：皮肤主题、本地通知、其他系统偏好。

### 已完成的后端骨架

后端目录：`backend/`

- AuthModule：注册、登录、刷新 token。
- UserModule：用户资料和偏好。
- SyncModule：个人数据同步、导出、软删除、后写胜出冲突策略。
- AiModule：AI 代理接口和使用日志。
- GroupModule：邀请制学习小组、邀请码、加入/退出、成员列表。
- ActivityModule：学习动态和积分事件。
- LeaderboardModule：个人榜、小组周榜、小组月榜。
- Prisma 数据模型和 Docker 部署配置。

注意：后端接口已具备基础能力，Flutter App 端也已加入云端同步、学习小组和排行榜 UI；下一步重点是真实账号体系、同步接口、小组动态和排行榜数据联调。

## 六、主要数据模型

| 模型 | 说明 |
|------|------|
| StudyTaskItem | 学习任务，含类型、课程、截止时间、状态、备注、提醒时间、子任务 |
| StudySubTaskItem | 子任务，含状态、截止时间、创建和更新时间 |
| StudyLogItem | 学习日志，含日期、课程、内容、问题、思考、下一步计划 |
| WeeklyReportItem | 周报，含时间范围、内容、来源日志 |
| StudyNote / NoteBlock | 学习笔记与块结构 |
| AiGeneratedLog | AI 生成的结构化学习日志 |
| AiTaskPlan | AI 拆解任务结果 |
| AiStudyAnalysis | AI 周报分析结果 |
| AiRiskWarning | AI 风险提醒 |
| AiFlashCard | AI 知识闪卡 |
| AiChatMessage | AI 聊天消息与会话 |
| AiConfig | 蓝心/DeepSeek 模型与参数配置 |

## 七、后端 API 摘要

- 账号：`POST /auth/register`、`POST /auth/login`、`POST /auth/refresh`
- 用户：`GET /me`、`PATCH /me/profile`
- 同步：`POST /sync/push`、`GET /sync/pull?cursor=...`、`GET /sync/export`
- 小组：`POST /groups`、`POST /groups/join`、`GET /groups/:id`
- 动态：`POST /activities`、`GET /groups/:id/activities`
- 排行榜：`GET /leaderboards/me`、`GET /leaderboards/groups/:id?range=week|month`
- AI：`POST /ai/study-log`、`POST /ai/task-plan`、`POST /ai/weekly-analysis`、`POST /ai/risk-warnings`、`POST /ai/flash-cards`、`POST /ai/chat`、`POST /ai/chat/stream`

## 八、后续开发建议

### 优先级高

1. Flutter 前端接入后端认证：登录、注册、刷新 token、用户资料同步。
2. 联调云端同步：本地数据 push/pull、软删除、同步状态与后端接口打通。
3. 将 AI 调用逐步迁移到后端代理，避免客户端直接暴露模型 Key。

### 优先级中

4. 学习小组 UI 接入真实后端：创建小组、邀请码加入、成员列表、组内动态。
5. 排行榜 UI 接入真实后端：个人总榜、小组周榜、小组月榜。
6. 后端接口错误态、离线态和额度超限态在 App 端的提示。

### 优先级低

7. 数据导出分享增强。
8. 多语言支持。
9. 比赛展示视频和演示数据脚本。
