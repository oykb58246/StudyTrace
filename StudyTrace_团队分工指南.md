# 《StudyTrace 学迹》团队策划文档分工指南

基于大赛官方《AIGC创新赛作品评分表》与《策划模版.pptx》，以下是为 4 人团队量身定制的任务认领清单。

## 👨‍💼 成员 A（产品经理）：统筹大局与商业价值撰写

**目标：** 拿下初赛 40% 权重的“应用价值”与部分“创新性”得分，把控整体 PPT 逻辑。

**具体任务（对应策划 PPT 模块）：**
* **【团队介绍】与【作品简介】：** 汇总团队成员信息与分工。撰写 200 字左右的作品概述，点明《StudyTrace》面向人群、应用场景及痛点。
* **【作品设计理念】：** 提炼核心创新点（主打“学习印记端侧化”及隐私保护）。阐述设计背景与洞察。确保理念贯穿整个作品。
* **【前景评估】：** 深度剖析用户需求程度（目标用户画像、核心痛点、高频应用场景、社会价值）。对比市面竞品，说明《StudyTrace》的市场欢迎程度及核心优势。

---

## 🎨 成员 B（UI与交互设计）：视觉呈现与交互体验撰写

**目标：** 负责门面担当，拿下评分表中的“产品界面”与“产品交互”得分。

**具体任务（对应策划 PPT 模块）：**
* **【作品宣传海报】：** 制作一张能够直观体现核心卖点的项目海报。
* **【作品的界面设计】：** 产出核心界面的效果图（如扁平化 Dashboard、vivo 蓝主题展示）并配文字说明。描述整体视觉风格及色彩体系背后的理由，确保界面完整美观。
* **【作品的交互设计】：** 选取 1-2 个核心操作流程（如：从专注计时结束到生成知识闪卡的闭环交互），用流程图或图文分步说明。重点介绍 Notion 风格块级交互等创新之处，体现交互的流畅与无断点。

---

## 💻 成员 C（前端 / 客户端开发）：技术落地与创新点撰写

**目标：** 证明 App 非套壳网页，拿下评分表中“可行性”与“产品功能”得分。

**具体任务（对应策划 PPT 模块）：**
* **【创新点说明】：** 详细阐释作品在功能和性能上的创新。重点说明如何利用 Flutter 3.x 架构实现跨平台，以及如何结合手机硬件（如调用摄像头拍照搜题、麦克风语音创建任务）。
* **【可行性论证】：** 结合代码实现进度，证明作品具备较强的技术可实现性，并阐明转化为实际应用的路径（如本地 JSON 存储架构如何为后续接驳端侧大模型做准备）。
* **材料准备：** 负责按要求录制/准备演示 Demo 及相关的产品原型文件。

---

## 🧠 成员 D（大模型 / 后端开发）：AI 引擎与模型应用撰写

**目标：** 拿下复赛 10% 权重的“大模型应用能力”得分，展示硬核 AI 技术深度。

**具体任务（对应策划 PPT 模块）：**
* **【大模型的具体应用说明】：** 详细撰写《StudyTrace》是如何应用大模型的。具体阐释对蓝心大模型（Chat Completions + Vision + OCR）的核心应用场景，并说明 DeepSeek 作为用户可选的自定义 API 接入能力。
* **技术机制拆解：** 深入描述核心 AI 功能是如何通过 API 调用实现的。例如：AI 分析型周报底层的结构化 Prompt 约束机制、流式对话的处理方式、蓝心多模态能力，以及用户自定义接入 DeepSeek 时的配置、测试与回退机制。
